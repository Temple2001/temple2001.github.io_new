# 목차

# 블로그를 교체하는 이유

예전 블로그는 다른 사람이 만든 탬플릿을 가져온거라 이것저것 잡다한 기능들이 껴있음(깃허브 댓글 기능, give me a coffee, SNS 공유 기능)

내가 만든게 아니라 블로그의 전체 구조를 파악하는데 어려움이 많음

블로그를 만들 당시에는 배포까지 매우 편리했지만 내 프론트 개발 능력에는 아무 도움이 되지 못한다고 판단

블로그 디자인이나 성능이 퇴보하는 한이 있어도 블로그를 내 힘만으로 처음부터 만드는 것이 옳다고 생각

# 기술 스택

기존 블로그에서 사용하던 Gatsby는 gatsby-plugin같은 유용한 기능들을 사용할 수 있고 github-pages로 배포되는 많은 블로그들이 사용하는 프레임워크이지만, Gatsby의 버전이 올라갈 때마다 생기는 각종 충돌 문제들 때문에 기존 블로그를 만들 때 애먹었던 적이 많음

따라서 Gatsby보다 더 유연성이 있고 커뮤니티도 더 큰 next.js를 사용하기로 결정

nextjs는 SSG(Static Site Generation)을 지원하기 때문에 github-pages에 배포하는 과정도 어려움이 없을 거라 생각

# generateStaticParams

오로지 build time에만 작동한다.

# Github Actions

actions/upload-pages-artifact(깃허브 공식 Action)를 사용

# dangerouslySetInnerHTML

다크모드 Flicker 현상 제거를 위해 사용

# useContext

Navigation 펼침 버튼 구현 및 Post 정보 Header에 출력하는 기능 구현을 위해 사용

# generateMetadata

Dynamic하게 Metadata를 구성하고 싶을 때 사용

# remark-unwrap-images plugin

mdx파일에서 `![alt](imagePath)` 형태로 작성하면 p태그 안에 img태그가 들어가는 상황이 발생하여 다양한 문제가 일어남
(나의 경우 MDXComponent로 스타일링을 할 때 p태그 안에 div태그가 들어가 오류가 나오는 상황이 일어남)
이때 remark-unwrap-images 라는 plugin으로 img태그가 p태그로 감싸여지지 않도록 하여 문제를 해결했음.
